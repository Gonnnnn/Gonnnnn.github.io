# 16234 인구 이동

### 군집 찾기

- visited를 통해 방문 여부를 표현하는 것은 필수이다.
- 방문 안된 애들을 기준으로 BFS를 실행한다. 이 때 군집에 포함될 수 있는지의 여부 등은 문제를 잘 이해하고 구현
- ***방문해봤다고해서 visited이면 안된다. queue에 포함 시키지 않을거면 visited로 해놓지 않아야 다른 군집에 포함될 여지가 남는다. 너무 당연한 말이지만 꼭 기억하자.***

### 시간 초과

- 함수를 불러오는 과정이 많아질 수록 시간이 더 쓰이는 것은 당연하다. ***논리가 완벽하다고 생각되면 당황하지 말고 함수화 했던 것들을 그냥 이어버리자.***

# 16236 아기상어

### 탐색 우선순위

- 탐색에 있어서 우선순위를 지정해야할 때가 있다.
- dy, dx에서 상하좌우 순서를 조정하는 것도 좋은 방법이다.
- 위 방법만으로 되지 않을 수 있다. 당연하다. 이럴 때는 탐색으로 후보군을 정하고 우선순위를 기준으로 거르는 반복문 등을 따로 구현할 필요가 있다. ***욕심내지 말고 이렇게 구현부터 완벽히 하자***

### Map 정보가 다 같은 BFS

- Map 정보가 고정되어있고, 단순히 해당 타겟의 좌표 등을 탐색할 때 구현은 쉽다.

```python
# q는 탐색할 좌표가 담긴 queue
while(q):
	itr = len(q)
	for i in range(itr):
		...
```

- 탐색 후 타겟이 결정되면, 바꿔야할 map 정보를 꼼꼼히 확인하고 잘 갱신하자

# 17143 낚시왕

### Class

- Map 내에서 여러 객체들이 움직인다고 해서, 꼭 클라스화 할필요까지는 없다.

### Map Update

- 뭐가 뭘 잡아먹고, 어쩌고 하는 다양한 조건이 있을 수 있다. 빈 Map을 새로 짠 후, 객체를 모두 이동시키고, 이동시키며 그 때 그 때 조건에 맞게 빈 Map을 채워나가는 식으로 하는게 기존 Map을 가지고 노는 것보다 간편하다. 일단 이렇게 시작하자. 너무 욕심내지 말자 이 부분에서
- 객체가 움직이는 것은 객체 내에서 함수화 하면 좋을 것 같다는 생각이 자연스래 들지만, 객체는 정보만 포함하도록 하자. 가야할 위치를 predict하는 것만 하도록 하자. 실제 Map내의 위치를 update하는 것은 외부에서 해주자. 이게 더 “좋은 방법”인지는 모르겠지만 훨씬 편하다.
